ScavApp

Project Summary

ScavApp is an application meant to connect friends with their surroundings. This application can help connect people to a new city, bring friends together, provide a fun way to exercise, and much more. This application provides users with the option of choosing from pre-made scavenger hunts or creating their own scavenger hunt. Users can search for pre-made scavenger hunts in all 50 states that are ready to play whenever they want. Or, if they are looking for a more personalized experience, they can take part in our guided process that facilitates the process of creating a scavenger hunt.

Design Process

1. Understand

Competitor Analysis & S.W.O.T Profile

Before creating my product, I needed to first research and understand the problem I was going to address. In order to determine this, I conducted a competitor analysis of GooseChase and Let’s Roam, the two leading scavenger hunt applications available, and summarized in their information into individual S.W.O.T profiles.

From this I developed a problem statement which, in turn, helped me to create a possible solution.

2. Observe

User Interviews & Surveys

Once the foundation was set, surveys and interviews were conducted in order to further understand the user’s needs and goals. The problem statement was then updated further coincide with the research findings listed below.

3. Point of View

Affinity Mapping

All of the responses from the user interviews were written down on sticky notes with each color representing a different interview participant. These sticky notes were grouped based on similar themes and titles were assigned to each group to summarize these ideas. This affinity mapping helped to understand trends in the thoughts and opinions of the potential user.

User Personas

User personas were created to bring the above findings to life and to help prioritize these needs. Three different personas were created to acknowledge that people will be using this app for different purposes.

4. Ideate

User Journeys

From these user personas, I developed user journeys to help visualize the steps the user will need to complete in order to accomplish their goals. By understanding the step by step processes as well as the users thoughts and emotions, designers can truly put themselves in the shoes of the users.

User Flows

Once I understood the goals and needs of the user, user flows were created to demonstrate all of the pages the user would need to work their way through to accomplish their goals. These created the foundation needed to begin developing the information architecture of the application.

Card Sorting & Sitemapping

Card sorting was performed to help understand the user’s point of view of how information should be grouped. This provided the structure for mapping the application. The information gained from the card sorting was used to develop a site map of how things will be grouped and structured in the application.

5. Prototype

Low, mid, and high fidelity wireframes were created using user journeys, user flows, and the sitemap. These wireframes were turned into prototypes to bring an element of interactivity and to bring the wireframes to life.

Low-Fidelity

Quick sketches were created to build a foundation of the core functionality of the app.

Mid-Fidelity

Mid-fidelity wireframes were created using the program Balsamiq. These were created to show more functionality than the low-fidelity wireframes but did not have the the level of detail found in the high-fidelity wireframes.

High-Fidelity

The last stage was to create high-fidelity wireframes using AdobeXD. These incorporated all of the functionality and also included the UI experience. These screens were mapped to create a prototype that was used for user testing.

6. Test

Usability Testing

Usability testing was performed using the high-fidelity prototype to identify pain points and areas of improvement before handing off the design to the developers. Users were read scenarios then asked to complete tasks using the prototype.

Affinity & Rainbow Mapping

The results from the usability tests were grouped together based on similar themes using affinity mapping. These themes were used in rainbow mapping to help illustrate the prominence of the problem and to consider possible solution for these problems. From this spreadsheet, I was able to prioritize which issues to fix.​​​​​​​

Iteration

After analyzing the results from the usability testing, changes were made to the prototype to address these pain points. Though these changes have been made there is always room for further iteration. Future iterations will include adding additional game types, providing more support while customizing games, adding animations, creating a mascot, polishing the UI design, and offering additional filter options.​​​​​​
